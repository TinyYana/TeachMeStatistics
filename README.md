# 統計解題駕訓班

這是給我這學期修的應用統計學的期末考前自己要用的互動頁。考前拿來臨時抱佛腳用的，不是要做成那種很大的遊戲化學習平台。

重點就放在最容易卡住的那幾個地方：

- 這題到底是哪一種題型
- 公式裡每個符號在裝什麼
- 數字要怎麼代進去
- 每一步為什麼這樣算
- 答錯的時候比較可能是哪一步歪掉

範圍先壓在常用、又適合拆開來互動的主題。一次做太大很容易變成「做得很漂亮、但根本不會打開來讀」的網站，所以先收著。

## 目前內容

- 首頁概念地圖（已補滿考試範圍）
- 題型判斷器
- 描述統計
- 機率基礎
- 二項分配
- 常態分配 Z 分數
- 錯題診斷基礎版
- 每章考卷型練習題

概念地圖這版照期末考目錄補齊了：條件機率、期望值與變異數、間斷／連續型、抽樣分配與中央極限定理、t/卡方/F 臨界值都進去了，先讓人看得到全貌跟先後順序。

分步計算器目前還是只做描述統計、機率、二項、常態這四個核心。信賴區間、假設檢定、還有上面新加的那幾個，先停在概念地圖跟題型辨識，計算器晚點再補。這是刻意留的範圍，不是漏做。

考試不考的就直接不放：貝氏定理、聯合與邊際機率、卜瓦松、指數分配、用常態近似二項。放了只是占版面。

## 學習設計

每章盡量照同一條動線走：

1. 這章在解決什麼問題
2. 看到什麼題目特徵時要用這章
3. 需要哪些前置概念
4. 最小例題
5. 公式與符號翻譯
6. 分步計算
7. 互動驗證
8. 題型辨識練習與錯題診斷
9. 考卷型練習
10. 解題模板

會這樣排其實沒什麼玄機：考試卡住通常不是完全不懂，是題型判斷、公式代入、或某個小步驟歪掉。

## 開發

```bash
npm install
npm run dev
```

如果 PowerShell 擋到 `npm.ps1`，在 Windows 上改用：

```bash
npm.cmd run dev
```

常用指令：

```bash
npm.cmd run test
npm.cmd run build
```

## 專案結構

```txt
index.html
chapters/
  descriptive.html
  probability.html
  binomial.html
  normal.html
src/
  content/       # 概念、題型、診斷、練習資料
  lib/           # 統計計算與圖表 helper
  pages/         # 各頁互動邏輯
  main.ts        # 共用導覽與互動 renderer
  styles.css
tests/
  lib.test.ts
```

## 驗證

主要靠 Vitest 跟 Vite build 顧著：

- 計算 helper 算得對不對
- 題型判斷資料完不完整
- 錯題診斷資料完不完整
- 每章有沒有考卷型練習
- production build 過不過
